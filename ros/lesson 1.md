Ноды и топики
 Любой исполняемый файл представляет собой ноду. Ноды общаются друг с другом с помощью некоторого эфира (топика). Ноды передают с помощью топика сообщения, вызывают сервисы и исполняют команды.
Нужно симулировать среду, в которой смогут существовать ноды 

ВЫПОЛНИТЬ:

	roscore

Запустить ноду turtlesim_node из  пакета turtlesim.

ВЫПОЛНИТЬ:

	rosrun turtlesim turtlesim_node

Запустить ноду turtlesim_teleop_key из  пакета turtlesim для управления роботом с клавиатуры с помощью стрелок.

ВЫПОЛНИТЬ:

	rosrun turtlesim turtlesim_teleop_key

Узнать какие ноды запущены

ВЫПОЛНИТЬ:

	rosnode list

rosout - нода-логгер, которая запускается автоматически. Она принимает все сообщения типа log и формирует файлы логов.

Создать еще один экран с той же самой нодой и управлять ими с клавиатуры.

ВЫПОЛНИТЬ:

	rosrun turtlesim turtlesim turtlesim_node__name:=turtlesim_node_second

В случае если будет введен запрос на запуск уже выполняющейся ноды с тем же именем, старая нода аварийно завершит выполнение.

Топик представляет собой связь "многие ко многим". Ноды могут публиковать сообщения в топике или могут быть подписанными на топик. Узнать информацию о ноде

ВЫПОЛНИТЬ:

	rosnode info /turtlesim

Узнать тип топика, тип сообщений, которые приходят в топик, и какие ноды с ним связаны:

ВЫПОЛНИТЬ:

	rostopic info /turtlesim/cmd_vel


Узнать о типе топика:

rosmsg show geometry_msgs/Twist

Послать сообщение в топик с помощью клавиатуры:

ВЫПОЛНИТЬ:

	rostopic pub /turtlesim/smd_vel geometry_msgs/Twist

Отследить, какие сообщения приходят в топик:

ВЫПОЛНИТЬ:

	rostopic echo /turtlesim/smd_vel

В ROS можно построить визуальный граф отношений между нодами.

ВЫПОЛНИТЬ:

	rosrun rqt_graph rqt_graph

Сервис в ROS - это сообщение, посылаемое одной конкретной нодой другой конкретной ноде. 
Узнать, какие сервисы предоставляются системе и информацию об определенном сервисе: 

ВЫПОЛНИТЬ:

	rosservice list
	rosservice info clear

Вызвать сервис:

ВЫПОЛНИТЬ:

	rosservice call clear
